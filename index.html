Back to basics

  • The fundamental unit of work in git is the commit
  • Performing good commits is the foundation of effective use of git. Good commits:
    • easier for you to track your own work
    • easier for others to review your work
    • easier to save you from yourself
    • easier for others to save the project from your horrific mistakes
    • help you develop a more focused work flow

Good Commits ™

All this talk about good commits... what do they look like?
  • Keep commits small
  • Group related changes into the same commit
  • Keep unrelated changes out of that commit
  • WRITE A USEFUL COMMIT MESSAGE

Make better work flows

  • Break each task in your project down into a series of small changes
  • Work on one change at a time until (you think) you get that change working
  • Commit that one change
  • WRITE A USEFUL COMMIT MESSAGE

Git can help

Git has loads of tools to help you perform good commits
  • View unstaged files and changes (git-status,git-diff)
  • View staged changes (git-diff)
  • Unstage changes (git-reset)
  • Stage specific lines in a changed file (git-add)

Beyond the fundamentals

Good commits make more advanced tasks easier
  • handling merge conflicts (git-merge)
  • finding and reverting bad changes (git-bisect/git-revert)
  • cherry picking commits (git-cherry-pick)
  • searching history for particular changes (git-grep)
